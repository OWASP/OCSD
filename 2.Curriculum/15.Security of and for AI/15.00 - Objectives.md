# Objectives for Module 15 - Security of and for AI

## Learning Objectives

After completing this module, developers will be able to:

- Identify and mitigate OWASP LLM Top 10 (2025) vulnerabilities
- Implement defenses against prompt injection and system prompt leakage
- Secure vector databases and RAG implementations against embedding attacks
- Apply controls for excessive agency, unbounded consumption, and misinformation risks
- Design guardrails and content filtering for AI-generated outputs

---

## Knowledge

Developers will obtain the following knowledge, upon completion of this module. 

- The general architecture of modern AI systems
- Security requirements by category


## Skills

Developers will learn the following practical skills, upon completion of this module. 

- The basics of threat modeling AI systems
- How to perform basic checks during development and testing of AI systems


## Competencies

This module is designed to help learners develop the intuition requirements required to do the following.   

- Develop security knowledge bases and checklists to implement when designing AI systems, mapping risks and defenses to full architectures
- Integrate it into CI/CD pipelines, code reviews, and tests during development
- Apply it as a verification framework for assessments, such as penetration testing and audits
- Reference specific requirements when evaluating AI vendors and third-party models during procurement
